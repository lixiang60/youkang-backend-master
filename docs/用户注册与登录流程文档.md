# 用户注册与登录流程文档

---

## 一、注册流程

### 接口信息

| 项目 | 说明 |
|------|------|
| 请求方式 | `POST` |
| 接口路径 | `/register` |
| 权限要求 | 无（匿名访问） |
| Content-Type | `application/json` |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| username | String | 是 | 用户名（2~20个字符） |
| password | String | 是 | 密码（5~20个字符） |
| code | String | 视配置 | 验证码（验证码开关开启时必填） |
| uuid | String | 视配置 | 验证码唯一标识（配合 code 使用） |

### 请求示例

```json
{
  "username": "testuser",
  "password": "123456",
  "code": "a3b2",
  "uuid": "c1d2e3f4-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
}
```

### 注册流程图

```
客户端                         服务端
  |                              |
  |  POST /register              |
  |----------------------------->|
  |                              |
  |                   检查注册开关（sys_config: sys.account.registerUser）
  |                              |── 关闭 ──> 返回 "当前系统没有开启注册功能！"
  |                              |
  |                   验证码校验（如果开启）
  |                              |── Redis 取验证码失败 ──> 抛出 CaptchaExpireException
  |                              |── 验证码不匹配 ──> 抛出 CaptchaException
  |                              |
  |                   参数校验
  |                              |── 用户名为空 ──> 返回 "用户名不能为空"
  |                              |── 密码为空 ──> 返回 "用户密码不能为空"
  |                              |── 用户名长度不合法 ──> 返回 "账户长度必须在2到20个字符之间"
  |                              |── 密码长度不合法 ──> 返回 "密码长度必须在5到20个字符之间"
  |                              |── 用户名已存在 ──> 返回 "保存用户'xxx'失败，注册账号已存在"
  |                              |
  |                   执行注册
  |                              |── BCrypt 加密密码
  |                              |── 插入 sys_user 表
  |                              |── 异步记录注册日志（sys_logininfor）
  |                              |
  |          注册成功             |
  |<-----------------------------|
  |                              |
```

### 响应示例

**成功：**

```json
{
  "code": 200,
  "msg": "操作成功"
}
```

**失败：**

```json
{
  "code": 500,
  "msg": "保存用户'testuser'失败，注册账号已存在"
}
```

### 注册开关

注册功能通过 `sys_config` 表的配置项控制：

| 配置键 | 说明 | 值 |
|--------|------|----|
| `sys.account.registerUser` | 注册功能开关 | `true` 开启 / `false` 关闭 |

可通过系统管理 → 参数设置页面修改，或直接更新数据库：

```sql
UPDATE sys_config SET config_value = 'true' WHERE config_key = 'sys.account.registerUser';
```

### 验证码机制

1. 前端先调用 `GET /captchaImage` 获取验证码图片和 uuid
2. 验证码答案存入 Redis，key 为 `captcha_codes:{uuid}`，有有效期
3. 注册时携带 `code`（用户输入）和 `uuid`（获取验证码时返回的）
4. 服务端从 Redis 取出答案对比，验证后删除

验证码开关通过 `sys_config` 表控制：

| 配置键 | 说明 | 值 |
|--------|------|----|
| `sys.account.captchaEnabled` | 验证码开关 | `true` 开启 / `false` 关闭 |

### 注册后用户状态

注册成功后用户会被写入 `sys_user` 表，默认状态：

| 字段 | 值 | 说明 |
|------|----|------|
| user_name | 用户输入的用户名 | |
| nick_name | 同 user_name | |
| password | BCrypt 加密后的密码 | |
| pwd_update_date | 注册时间 | |
| status | `0` | 正常状态 |
| dept_id | 无 | 未分配部门 |
| role_ids | 无 | 未分配角色 |

> 注册后用户没有角色和部门，需要管理员手动分配权限。

### 相关代码

| 文件 | 说明 |
|------|------|
| `youkang-admin/controller/system/SysRegisterController.java` | 注册接口入口 |
| `youkang-framework/web/service/SysRegisterService.java` | 注册核心逻辑 |
| `youkang-system/service/impl/SysUserServiceImpl.java` | `registerUser()` — 插入用户 |
| `youkang-common/core/domain/model/RegisterBody.java` | 注册请求体（继承 LoginBody） |
| `youkang-framework/config/SecurityConfig.java` | `/register` 路径放行配置 |

## 二、登录流程

### 接口信息

| 项目 | 说明 |
|------|------|
| 请求方式 | `POST` |
| 接口路径 | `/login` |
| 权限要求 | 无（匿名访问） |
| Content-Type | `application/json` |

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| username | String | 是 | 用户名 |
| password | String | 是 | 密码 |
| code | String | 视配置 | 验证码 |
| uuid | String | 视配置 | 验证码唯一标识 |

### 请求示例

```json
{
  "username": "admin",
  "password": "admin123",
  "code": "a3b2",
  "uuid": "c1d2e3f4-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
}
```

### 登录流程图

```
客户端                         服务端
  |                              |
  |  POST /login                 |
  |----------------------------->|
  |                              |
  |                   验证码校验（如果开启）
  |                              |── Redis 取验证码失败 ──> 抛出 CaptchaExpireException
  |                              |── 验证码不匹配 ──> 抛出 CaptchaException
  |                              |
  |                   登录前置校验
  |                              |── 用户名/密码为空 ──> 抛出 UserNotExistsException
  |                              |── 用户名/密码长度不合法 ──> 抛出 UserPasswordNotMatchException
  |                              |── IP 在黑名单中 ──> 抛出 BlackListException
  |                              |
  |                   Spring Security 认证
  |                              |── AuthenticationManager.authenticate()
  |                              |── UserDetailsService.loadUserByUsername()
  |                              |── 密码校验失败 ──> 抛出 UserPasswordNotMatchException
  |                              |
  |                   认证成功
  |                              |── 记录登录IP和登录时间
  |                              |── 生成 JWT Token
  |                              |── 异步记录登录日志（sys_logininfor）
  |                              |
  |    返回 token                |
  |<-----------------------------|
  |                              |
```

### 响应示例

**成功：**

```json
{
  "code": 200,
  "msg": "操作成功",
  "token": "eyJhbGciOiJIUzI1NiJ9.eyJsb2dpbl91c2VyX2tle..."
}
```

**失败：**

```json
{
  "code": 401,
  "msg": "用户密码错误"
}
```

### Token 使用

登录成功后，后续请求在 Header 中携带 Token：

```
Authorization: Bearer {token}
```

Token 有效期 30 分钟，支持自动续期（距过期不足20分钟时刷新）。

### IP 黑名单

通过 `sys_config` 表配置：

| 配置键 | 说明 | 示例值 |
|--------|------|--------|
| `sys.login.blackIPList` | 禁止登录的IP列表 | `192.168.1.100,10.0.0.*` |

### 相关代码

| 文件 | 说明 |
|------|------|
| `youkang-admin/controller/system/SysLoginController.java` | 登录接口入口 |
| `youkang-framework/web/service/SysLoginService.java` | 登录核心逻辑（认证 + Token 生成） |
| `youkang-framework/web/service/TokenService.java` | JWT Token 管理（创建、验证、刷新） |
| `youkang-framework/security/filter/JwtAuthenticationTokenFilter.java` | JWT 认证过滤器 |
| `youkang-framework/security/handle/AuthenticationEntryPointImpl.java` | 认证失败处理 |
| `youkang-framework/config/SecurityConfig.java` | `/login` 路径放行配置 |

## 三、认证机制概览

```
                        ┌─────────────┐
                        │   客户端     │
                        └──────┬──────┘
                               │
                 POST /login   │   POST /register
                 POST /captcha │
                               │
                        ┌──────▼──────┐
                        │ Spring       │
                        │ Security     │  ← /login, /register, /captchaImage 放行
                        │ Filter Chain │  ← 其他请求需携带 JWT Token
                        └──────┬──────┘
                               │
                  ┌────────────┼────────────┐
                  │            │            │
           /login         /register    其他请求
                  │            │            │
           登录认证        注册用户     JWT Filter 校验
                  │            │            │
           返回 Token      返回成功    通过 → 放行
                                        失败 → 401
```

### 安全配置要点

| 配置 | 说明 |
|------|------|
| CSRF | 禁用（前后端分离，不使用 Session） |
| Session | STATELESS（无状态，基于 Token） |
| 密码加密 | BCrypt |
| Token 有效期 | 30 分钟 |
| Token 自动续期 | 距过期不足 20 分钟时刷新 |
