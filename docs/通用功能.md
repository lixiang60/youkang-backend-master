# 通用功能

## 文件上传

系统支持多种存储方式，包括本地存储、阿里云OSS、MinIO，通过配置可灵活切换。

### 配置说明

在 `application.yml` 中配置：

```yaml
# 云存储配置
cloud-storage:
  # 存储类型：local（本地）、oss（阿里云）、minio（MinIO）
  type: ${STORAGE_TYPE:local}
  # 访问域名（前端访问文件时使用）
  domain: ${STORAGE_DOMAIN:http://localhost:3564}
  # 阿里云OSS配置
  oss:
    endpoint: ${OSS_ENDPOINT:oss-cn-hangzhou.aliyuncs.com}
    access-key-id: ${OSS_ACCESS_KEY_ID:}
    access-key-secret: ${OSS_ACCESS_KEY_SECRET:}
    bucket-name: ${OSS_BUCKET_NAME:youkang}
  # MinIO配置
  minio:
    endpoint: ${MINIO_ENDPOINT:http://127.0.0.1:9000}
    access-key: ${MINIO_ACCESS_KEY:minioadmin}
    secret-key: ${MINIO_SECRET_KEY:minioadmin}
    bucket-name: ${MINIO_BUCKET_NAME:youkang}
```

### 存储类型

| 类型 | 说明 | 适用场景 |
|------|------|---------|
| `local` | 本地文件系统存储 | 开发环境、单机部署 |
| `oss` | 阿里云对象存储OSS | 生产环境、分布式部署 |
| `minio` | MinIO对象存储 | 私有云、内网环境 |

### 使用方式

#### 本地存储（默认）

```yaml
cloud-storage:
  type: local
  domain: http://localhost:3564
```

文件存储路径由 `youkang.profile` 配置，默认为 `D:/youkang/uploadPath`

#### 阿里云OSS

**方式一：修改配置文件**

```yaml
cloud-storage:
  type: oss
  domain: https://your-domain.com
  oss:
    endpoint: oss-cn-hangzhou.aliyuncs.com
    access-key-id: 你的AccessKeyID
    access-key-secret: 你的AccessKeySecret
    bucket-name: your-bucket
```

**方式二：环境变量（推荐生产环境）**

```bash
STORAGE_TYPE=oss \
STORAGE_DOMAIN=https://your-domain.com \
OSS_ENDPOINT=oss-cn-hangzhou.aliyuncs.com \
OSS_ACCESS_KEY_ID=你的AccessKeyID \
OSS_ACCESS_KEY_SECRET=你的AccessKeySecret \
OSS_BUCKET_NAME=your-bucket \
java -jar youkang-admin.jar
```

#### MinIO

**方式一：修改配置文件**

```yaml
cloud-storage:
  type: minio
  domain: http://minio.your-domain.com
  minio:
    endpoint: http://192.168.1.100:9000
    access-key: minioadmin
    secret-key: minioadmin
    bucket-name: youkang
```

**方式二：环境变量**

```bash
STORAGE_TYPE=minio \
STORAGE_DOMAIN=http://minio.your-domain.com \
MINIO_ENDPOINT=http://192.168.1.100:9000 \
MINIO_ACCESS_KEY=minioadmin \
MINIO_SECRET_KEY=minioadmin \
MINIO_BUCKET_NAME=youkang \
java -jar youkang-admin.jar
```

### API接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/system/file/upload` | POST | 单文件上传 |
| `/system/file/uploads` | POST | 批量文件上传 |
| `/system/file/download/{fileId}` | GET | 文件下载 |
| `/system/file/preview/{fileId}` | GET | 文件预览 |
| `/system/file/{fileId}` | GET | 获取文件信息 |
| `/system/file/list` | GET | 文件列表查询 |
| `/system/file/{fileIds}` | DELETE | 删除文件 |

### 上传示例

**单文件上传**

```bash
curl -X POST "http://localhost:3564/system/file/upload" \
  -H "Authorization: Bearer {token}" \
  -F "file=@/path/to/file.pdf" \
  -F "businessId=ORDER_001"
```

**批量上传**

```bash
curl -X POST "http://localhost:3564/system/file/uploads" \
  -H "Authorization: Bearer {token}" \
  -F "files=@/path/to/file1.pdf" \
  -F "files=@/path/to/file2.pdf" \
  -F "businessId=ORDER_001"
```

### 注意事项

1. **文件大小限制**：默认单文件最大 10MB，总请求最大 20MB，可在 `application.yml` 中修改
2. **支持的文件类型**：默认支持常见文件类型，可在 `MimeTypeUtils` 中扩展
3. **切换存储类型**：只需修改 `cloud-storage.type` 配置，无需修改代码
4. **域名配置**：`domain` 用于生成文件访问URL，请根据实际部署环境配置
